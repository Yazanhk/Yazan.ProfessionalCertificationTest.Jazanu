SECTION 1: QUESTIONS WITH TWO CORRECT ANSWERS
Question 1: Which TWO of the following are characteristics of a hot site disaster recovery strategy? (Select 2)
A) Lowest cost recovery option
B) Fully operational with real-time data mirroring
C) Takes weeks to become operational
D) Near-instantaneous failover capability
E) Requires hardware installation after disaster
F) Data synchronized periodically, not real-time
Correct Answers: B and D

A hot site is a fully operational backup facility with identical hardware/software and real-time data synchronization. It provides near-instantaneous failover with minimal data loss. Cold sites (A, C, E) are basic facilities requiring hardware installation and taking days/weeks to become operational. Warm sites (F) have periodic synchronization and take hours to days.
Question 2: Which TWO security control types operate BEFORE an attack occurs? (Select 2)
A) Detective
B) Preventive
C) Corrective
D) Deterrent
E) Compensating
F) Directive
Correct Answers: B and D

Preventive controls physically or logically restrict unauthorized access before an attack occurs. Deterrent controls psychologically discourage intrusions before they happen. Detective (A) identifies attacks during occurrence, Corrective (C) operates after an attack, compensating (E) substitutes for principal controls, and Directive (F) enforces rules of behavior.
Question 3: Which TWO of the following are secure alternatives to Telnet? (Select 2)
A) SSH
B) RDP
C) FTP
D) SFTP
E) HTTP
F) SNMPv1
Correct Answers: A and D

SSH (Secure Shell) provides encrypted remote command-line access as a secure replacement for Telnet. SFTP (SSH File Transfer Protocol) provides secure file transfer encryption. RDP (B) is for remote desktop, not Telnet replacement. FTP (C) and HTTP (E) are insecure. SNMPv1 (F) uses cleartext community strings.
Question 4: Which TWO of the following are characteristics of a decentralized computing architecture? (Select 2)
A) Single point of failure risk
B) Data processing distributed across multiple locations
C) All users rely on a central server
D) Blockchain is an example
E) Easier to secure than centralized
F) Lower latency than centralized always
Correct Answers: B and D

Decentralized computing distributes data processing and storage across multiple locations/devices. Blockchain is a key example using peer-to-peer distributed ledgers. Centralized computing (A, C) has single points of failure and relies on central servers. Decentralized is not necessarily easier to secure (E) nor always lower latency (F).
Question 5: Which TWO authentication factors are classified as "something you have"? (Select 2)
A) Password
B) Smart card
C) Fingerprint scan
D) Hardware token
E) PIN number
F) Voice recognition
Correct Answers: B and D

Something you have refers to physical objects possessed by the user. Smart cards and hardware tokens (OTP fobs, security keys) are ownership factors. Passwords/PINs (A, E) are "something you know." Fingerprint (C) and voice recognition (F) are "something you are" biometric factors.
Question 6: Which TWO of the following are symmetric encryption algorithms? (Select 2)
A) RSA
B) AES
C) ECC
D) Blowfish
E) DSA
F) Diffie-Hellman
Correct Answers: B and D

AES (Advanced Encryption Standard) and Blowfish are symmetric encryption algorithms using the same key for encryption and decryption. RSA (A), ECC (C), DSA (E) are asymmetric algorithms using public/private key pairs. Diffie-Hellman (F) is a key exchange protocol.
Question 7: Which TWO statements about a non-credentialed vulnerability scan are correct? (Select 2)
A) It logs into the system with provided credentials
B) It simulates an external attacker perspective
C) It provides detailed patch level information
D) It has higher false positives than credentialed scans
E) It requires admin privileges to run
F) It can scan encrypted data at rest
Correct Answers: B and D

Non-credentialed scans are unauthenticated, simulating external attackers probing for public-facing vulnerabilities. They have higher false positive rates than credentialed scans. Credentialed scans (A, C, E) log in with credentials to provide detailed internal visibility and have lower false positives. Neither scan can view properly encrypted data at rest (F).
Question 8: Which TWO of the following are deception technologies used in cybersecurity? (Select 2)
A) Honeypot
B) Firewall
C) Honeytoken
D) IDS
E) VPN
F) SIEM
Correct Answers: A and C

Honeypots are decoy systems designed to lure attackers. Honeytokens are bogus credentials or data snippets that alert administrators when accessed. Firewalls (B), IDS (D), VPNs (E), and SIEM (F) are security tools but not deception technologies.
Question 9: Which TWO are components of the AAA framework in cybersecurity? (Select 2)
A) Authentication
B) Availability
C) Authorization
D) Accounting
E) Assurance
F) Auditing
Correct Answers: A and C (Note: Accounting is also correct)

AAA stands for Authentication (verifying identity), Authorization (determining access rights), and Accounting (tracking user activity). Availability (B) is part of CIA triad, not AAA.
Question 10: Which TWO of the following are benefits of using a CASB (Cloud Access Security Broker)? (Select 2)
A) Shadow IT discovery
B) Increased server processing power
C) Data Loss Prevention enforcement
D) Reduced cloud storage costs
E) Automatic code compilation
F) Hardware acceleration
Correct Answers: A and C

CASB provides visibility into cloud usage including unsanctioned (Shadow IT) apps and protects sensitive data through DLP (Data Loss Prevention) enforcement. CASB does not increase server power (B), reduce storage costs (D), compile code (E), or provide hardware acceleration (F).
SECTION 2: STANDARD MULTIPLE CHOICE (Domain 1: Security Concepts)
Question 11: Which principle of information security ensures that a user cannot deny having performed an action?
A) Confidentiality
B) Integrity
C) Availability
D) Non-repudiation
Correct Answer: D

Non-repudiation ensures that individuals cannot deny creating or modifying data. It uses digital signatures and logging to provide proof of actions. Confidentiality ensures only authorized reading, integrity ensures authorized modification, and availability ensures accessible information.
Question 12: A security control that uses cameras to record entry to a data center is classified as which type?
A) Managerial
B) Technical
C) Operational
D) Physical
Correct Answer: D

Physical controls are devices that mediate access to premises and hardware. Cameras, fences, locks, and security guards are physical controls. Managerial controls provide oversight, operational controls rely on people for implementation, and technical controls are implemented in systems/software.
Question 13: Which functional security control type substitutes for a principal control as a compliance measure?
A) Preventive
B) Detective
C) Compensating
D) Corrective
Correct Answer: C

Compensating controls substitute for principal controls when the primary control cannot be implemented, often for framework compliance. Preventive controls block access, detective controls identify intrusions, and corrective controls fix incidents.
Question 14: A security team implements a firewall rule to block all traffic from a malicious IP address. What type of control is this?
A) Detective
B) Corrective
C) Preventive
D) Deterrent
Correct Answer: C

Preventive controls physically or logically restrict unauthorized access before an attack occurs. The firewall blocking traffic prevents the attack from reaching the target. Detective controls identify attacks during occurrence, corrective controls fix incidents after, and deterrent controls psychologically discourage attackers.
Question 15: The Chief Information Security Officer (CISO) is reviewing security policies and procedures. This represents which security control category?

Managerial controls give oversight of systems, including policy creation, risk management, and governance. Technical controls are implemented in software/hardware, operational controls rely on people for implementation, and physical controls mediate access to premises.

Integrity ensures data is stored and transferred as intended and that any modification is authorized. Confidentiality ensures only authorized reading, availability ensures accessible information, and non-repudiation prevents denial of actions.

Deterrent controls psychologically discourage intrusions by warning potential attackers of consequences. Question 19: A gap analysis is performed to:

Question 20: The principle of least privilege means:
A) Users should have administrative access for all tasks
B) Users should have only the minimum permissions necessary
C) All users should have identical permissions
D) Permissions should never be changed

Least privilege ensures users and applications operate with only the minimum permissions necessary to perform their functions. This reduces attack surface and limits damage from compromised accounts. Administrative access (A) violates this principle, identical permissions (C) ignore role differences, and permissions must change (D) as roles evolve.

Corrective controls respond to and fix incidents after they occur, potentially preventing reoccurrence. Examples include patch management, system restoration, and incident response. Preventive blocks before, detective identifies during, and deterrent discourages before.

Domain 2: Threat Actors and Vectors
Question 36: An attacker with low technical skill who uses pre-written scripts to launch attacks is called a:
A) White hat hacker
B) Script kiddie
C) Nation-state actor
D) Advanced Persistent Threat
Correct Answer: B

Script kiddies are unskilled attackers who rely on commodity tools and pre-written scripts rather than developing their own exploits. White hats (A) are authorized ethical hackers, nation-state actors (C) have high capability, and APTs (D) are sophisticated persistent threats.
Question 37: Which threat actor is most likely to be sponsored by a government for espionage purposes?
A) Script kiddie
B) Hacktivist
C) Nation-state actor
D) Organized crime
Correct Answer: C

Nation-state actors are attached to military/secret services, have high capability, and conduct espionage for strategic advantage. Script kiddies (A) are unskilled, hacktivists (B) are politically motivated but not government-sponsored, and organized crime (D) is financially motivated.
Question 38: A watering hole attack involves:
A) Sending phishing emails to executives
B) Compromising a website frequently visited by the target
C) Installing malware on USB drives
D) Using default credentials on network devices
Correct Answer: B

A watering hole attack compromises a third-party site that the threat actor knows the target uses, then infects visitors to that site. Spear phishing (A) targets individuals directly, drop attacks (C) use USB drives, and default credentials (D) are a misconfiguration.
Question 39: Which attack vector involves sending fraudulent messages via text messaging?
A) Vishing
B) SMiShing
C) Phishing
D) Pharming
Correct Answer: B

SMiShing (SMS phishing) uses text messaging to trick targets into using malicious resources. Vishing (A) uses voice channels, phishing (C) typically uses email, and pharming (D) uses DNS spoofing.
Question 40: An insider threat where an employee accidentally shares sensitive data via email is classified as:
A) Malicious internal threat
B) Unintentional insider threat
C) External threat
D) Nation-state threat
Correct Answer: B

Unintentional insider threats result from weak policies, lack of training, or human error without malicious intent. Malicious internal threats (A) intentionally cause harm, external threats (C) come from outside, and nation-state threats (D) are government-sponsored.
Question 41: What is typosquatting?
A) Creating fake social media profiles
B) Registering domain names similar to legitimate sites with common typos
C) Sending mass phishing emails
D) Installing keyloggers on public computers
Correct Answer: B

Typosquatting registers domain names that are common misspellings of legitimate domains, tricking users who mistype URLs. Fake profiles (A), phishing emails (C), and keyloggers (D) are different attack types.
Question 42: A false flag operation in cyber espionage refers to:
A) Using legitimate credentials
B) Making an attack appear to come from a different source
C) Encrypting all traffic
D) Using only open-source tools
Correct Answer: B

False flag operations disguise the true origin of an attack to mislead attribution, often making it appear to come from another group or nation. Legitimate credentials (A), encryption (C), and open-source tools (D) are not false flag operations.
Question 43: Which attack uses DNS spoofing to redirect users to malicious websites without their knowledge?
A) Phishing
B) Vishing
C) Pharming
D) SMiShing
Correct Answer: C

Pharming redirects users by DNS spoofing, poisoning DNS caches, or modifying host files. Users are redirected without clicking a link. Phishing (A) uses deceptive messages, vishing (B) uses voice, and SMiShing (D) uses text messages.
Question 44: A Business Email Compromise (BEC) attack typically:

BEC attacks pose as trusted parties (colleagues, partners, vendors) to trick victims into transferring funds or revealing sensitive information. Ransomware (A), software exploits (C), and physical breaches (D) are different attack types.

Hacktivists are motivated by political or ideological causes, conducting attacks to promote their agenda or protest. Organized crime (A) seeks profit, script kiddies (B) seek notoriety, and competitors (D) seek business advantage.
Question 46: An attack surface includes all EXCEPT:

The attack surface includes points where attackers can discover/exploit vulnerabilities: physical, network, application, and human surfaces. Company stock price (D) is not an attack surface as it cannot be directly exploited for system access.

Question 51: Which threat actor is most likely to use ransomware for financial extortion?

Shadow IT refers to systems, devices, software, or services used within an organization without explicit approval or oversight from IT/security departments. It is not about timing (A), encryption (C), or physical security (D).

Domain 3: Cryptography
Question 61: Which encryption algorithm uses the same key for both encryption and decryption?
A) RSA
B) AES
C) ECC
D) DSA
Correct Answer: B

AES (Advanced Encryption Standard) is a symmetric encryption algorithm using the same secret key for encryption and decryption. RSA (A), ECC (C), and DSA (D) are asymmetric algorithms using public/private key pairs.
Question 62: What is the primary purpose of a digital signature?
A) Encryption of large data files
B) Providing integrity, authentication, and non-repudiation
C) Key exchange between parties
D) Password storage
Correct Answer: B

Digital signatures provide integrity (detecting modification), authentication (verifying source), and non-repudiation (preventing denial). They are not efficient for bulk encryption (A), key exchange (C) is different, and password storage (D) uses hashing.
Question 63: Which hash algorithm produces a 128-bit digest and is now considered cryptographically broken?
A) SHA-256
B) MD5
C) SHA-512
D) bcrypt
Correct Answer: B

MD5 produces a 128-bit hash and has known collision vulnerabilities making it unsuitable for security. SHA-256 (A) produces 256 bits, SHA-512 (C) produces 512 bits, and bcrypt (D) is a key derivation function.
Question 64: A digital certificate contains a subject's public key and is issued by a:
A) Certificate Authority (CA)
B) Registration Authority (RA)
C) Key Distribution Center (KDC)
D) Domain Controller (DC)
Correct Answer: A

Certificate Authorities (CAs) issue digital certificates after verifying subject identity. RAs (B) assist with verification, KDCs (C) issue Kerberos tickets, and Domain Controllers (D) manage Windows domains.
Question 65: Which asymmetric encryption algorithm is based on the difficulty of factoring large prime numbers?
A) ECC
B) AES
C) RSA
D) Blowfish
Correct Answer: C

RSA security relies on the difficulty of factoring the product of two large prime numbers. ECC (A) uses elliptic curve mathematics, AES (B) and Blowfish (D) are symmetric algorithms.
Question 66: What is the purpose of salting passwords?
A) To encrypt the password twice
B) To add random data to prevent rainbow table attacks
C) To compress the password for storage
D) To convert the password to uppercase
Correct Answer: B

Salting adds a random unique value to each password before hashing, preventing pre-computed rainbow table attacks. It does not encrypt twice (A), compress (C), or convert case (D).
Question 67: Which version of TLS is the current standard offering the strongest security?
A) SSL 3.0
B) TLS 1.0
C) TLS 1.2
D) TLS 1.3
Correct Answer: D

TLS 1.3 (2018) is the latest, most secure version, removing support for older insecure algorithms and speeding up the handshake. SSL 3.0 (A) and TLS 1.0 (B) are deprecated, TLS 1.2 (C) is still used but being replaced.
Question 68: Perfect Forward Secrecy (PFS) ensures that:
A) Passwords are never transmitted
B) Compromised long-term keys cannot decrypt past sessions
C) All traffic is encrypted twice
D) Keys never expire
Correct Answer: B

PFS ensures that even if an attacker compromises the long-term private key, they cannot decrypt previously captured sessions because each session uses unique ephemeral keys. Passwords (A) can still be transmitted, double encryption (C) is not PFS, and keys should expire (D).
Question 69: Which type of encryption is best suited for bulk encryption of large amounts of data?
A) Asymmetric encryption
B) Symmetric encryption
C) Hashing
D) Digital signatures
Correct Answer: B

Symmetric encryption is fast and efficient for bulk encryption of large data volumes. Asymmetric encryption (A) is computationally expensive, hashing (C) is one-way, and digital signatures (D) provide authentication, not bulk encryption.
Question 70: A Trusted Platform Module (TPM) is a:
A) Software-based password manager
B) Cryptoprocessor for secure key storage
C) Network firewall appliance
D) Type of encryption algorithm
Correct Answer: B

TPM is a hardware cryptoprocessor implemented on CPU/motherboard for secure key generation and storage. It is not software (A), network device (C), or algorithm (D).
Question 71: Which EAP method uses digital certificates for both client and server authentication?
A) EAP-TLS
B) PEAP
C) EAP-FAST
D) LEAP
Correct Answer: A

EAP-TLS requires digital certificates for both client and server, providing the strongest security. PEAP (B) uses server certificates only, EAP-FAST (C) uses PACs, and LEAP (D) is Cisco-proprietary with vulnerabilities.
Question 72: What is the primary difference between SSL and TLS?
A) SSL is faster than TLS
B) TLS is the modern, secure replacement for SSL
C) SSL uses asymmetric encryption, TLS uses symmetric

Correct Answer: B

SBOM is an ingredient list for software, identifying components, libraries, and dependencies. Hardware (A), salaries (C), and bandwidth (D) are not SBOM content.
Question 195: Memory injection attacks are considered "fileless" because:
A) They delete files after execution
B) Malicious code exists only in RAM memory spaces without triggering disk write security scans
C) They use text files only
D) They run without operating systems
Correct Answer: B

Fileless malware executes directly in memory (RAM), hijacking legitimate processes to avoid leaving trace artifacts on physical storage disks.
Question 196: Which web vulnerability forces an authenticated user to perform unwanted actions?
A) XSS
B) CSRF
C) SQL injection
D) Buffer overflow
Correct Answer: B

CSRF (Cross-Site Request Forgery) forces authenticated users to execute unwanted actions on web applications. XSS (A) injects scripts, SQL injection (C) targets databases, and buffer overflow (D) is memory corruption.
Question 197: A race condition vulnerability occurs when:
A) Multiple threads access shared data with timing-dependent outcomes
B) Password is too short
C) Firewall is misconfigured
D) Encryption key is weak
Correct Answer: A

Race conditions occur when multiple threads/processes access shared data simultaneously and outcome depends on uncontrollable timing. Password length (B), firewall config (C), and weak keys (D) are different vulnerabilities.
Question 198: What is the primary difference between penetration testing and vulnerability scanning?
A) Scanning is automated; pen testing involves manual exploitation
B) Scanning is more expensive
C) Pen testing cannot find vulnerabilities
D) Scanning requires special hardware
Correct Answer: A

Vulnerability scanning is automated identification; penetration testing involves manual/semi-automated exploitation to prove impact. Scanning is generally cheaper (B), pen testing finds vulnerabilities (C), and scanning doesn't require special hardware (D).
Question 199: Which vulnerability is associated with end-of-life (EOL) systems?
A) They receive regular security updates
B) Vendor no longer provides security patches
C) They are always faster
D) They have better compatibility
Correct Answer: B

EOL systems no longer receive security patches from vendors, leaving them permanently exposed to new vulnerabilities. They don't receive updates (A), are not faster (C), and have worse compatibility (D).
Question 200: A bug bounty program:
A) Pays hackers to find vulnerabilities ethically
B) Charges for security patches
C) Rewards employees for attendance
D) Sells vulnerabilities to attackers
Correct Answer: A

Bug bounty programs pay ethical hackers cash rewards for reporting vulnerabilities. They don't charge for patches (B), attendance rewards (C) are different, and selling to attackers (D) is black hat market activity.

Question 201: Which authentication protocol uses tickets to verify identity inside an Active Directory environment?
A) RADIUS
B) TACACS+
C) Kerberos
D) SAML
Correct Answer: C

Kerberos relies on Key Distribution Centers (KDC) to issue Tickets to clients, enabling secure authentication over insecure network pathways inside Windows environments.
Question 202: What is the purpose of a CASB in cloud security?
A) Increase cloud storage
B) Enforce security policies between users and cloud apps
C) Provide cloud computing power
D) Backup cloud data
Correct Answer: B

CASB (Cloud Access Security Broker) enforces security, compliance, and governance policies between cloud service users and cloud applications. Storage (A), computing (C), and backup (D) are different functions.
Question 203: A buffer overflow attack overwrites which critical memory location?
A) Stack pointer only
B) Return address to execute malicious code
C) Heap only
D) Data segment only
Correct Answer: B

Buffer overflow overwrites the return address to redirect execution to malicious shellcode. While it may affect stack (A), heap (C), or data (D), the primary hijack targets the return address pointer.
Question 204: Which wireless encryption standard uses GCMP-256 for government grade security?
A) WPA2-Personal
B) WPA3-Enterprise
C) WEP
D) WPA
Correct Answer: B

WPA3-Enterprise offers 192-bit security suite components using GCMP-256 to deliver highly protected military and government grade operations. WPA2-Personal (A) uses AES-128, WEP (C) and WPA (D) are broken.
Question 205: Containerization differs from hardware virtualization because containers share:
A) The host operating system kernel
B) Virtual hard drives only
C) Network routing tables only
D) Application configuration scripts
Correct Answer: A

Containers run as isolated application structures directly on the host OS kernel, making them lightweight and fast. Full virtual machines require installing full individual guest OS environments.
Question 206: Infrastructure as Code (IaC) allows engineers to:
A) Manually plug in servers
B) Define and provision data center structures using machine-readable configuration code files
C) Track physical network cables
D) Build mobile web applications
Correct Answer: B

IaC allows administrators to programmatically provision and automate networks, firewalls, and servers through configuration configuration templates (such as Terraform blueprints), ensuring consistency.
Question 207: What is the principal purpose of fuzzing tools during software security assessments?
A) To optimize system database performance speeds
B) Inputting random, malformed data streams into fields to force software crashes and discover bugs
C) Compressing code files down into binary executables
D) Designing aesthetic user interface components
Correct Answer: B

Fuzzing injects randomized invalid anomalies into software entry vectors to pinpoint memory allocation gaps, input filters, or edge-case handling vulnerabilities.
Question 208: Code signing provides:
A) Code obfuscation only
B) Authenticity and integrity verification of software
C) Runtime performance improvement
D) Memory optimization
Correct Answer: B

Code signing uses cryptographic digital signatures to verify that software comes from a known trusted publisher and hasn't been altered by malicious third parties.
